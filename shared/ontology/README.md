# Nostra Core Ontology

This directory holds the minimal Nostra-native ontology layer for graph semantics.

## Contract

- The ontology is intentionally small and additive.
- The current four core relations are the leading v1 freeze candidate and must pass earned-freeze validation before being treated as final.
- Core terms:
  - `Space`
  - `Contribution`
  - `Relation`
  - `Capability`
  - `ProvenanceScope`
- Space-local extensions must be namespace-prefixed and additive.
- `ProvenanceScope` remains a closed core set aligned to `system`, `actor`, and `agent`.
- Breaking ontology changes require steward review and a compatibility note.

## Compatibility Rules

1. Additive terms are preferred.
2. Core terms must not be redefined by Space-local vocabularies.
3. Relation semantics should remain stable across versions.
4. Provenance scope naming should stay aligned with `shared/specs.md`.
5. Space-local ontologies must not extend `ProvenanceScope`.
6. The core ontology is validated against reference comparators before v1 freeze is earned.
