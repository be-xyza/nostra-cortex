# Research Initiative: Nostra Development Space ("Nostra Prime")

## 1. Core Objective
To design and strategize the **Nostra Development Space** (also referred to as "Nostra Prime"), this is the meta-instance of the Nostra platform used to build Nostra itself. We treat development as the primary use case, using the "In-IDE Antigravity" process and BMAD framework as our playback model.

## 2. The Concept: "Nostra Prime"
**"The Map is the Territory."**
We are building the harness for our own development. This space is not just a project management tool; it is a **meta-instance** of the Nostra platform.

*   **Goal**: Log questions, ideas, research, issues, and schemas in a structured, graph-based format.
*   **Utility**: Use this system to plan, build, and test the **Workflow Builder** and **Cortex**.

## 3. Space Structure & Granularity (The Playbook)

We organize the space based on **Activity Domains**, which map directly to the **BMAD Agent Role Taxonomy**. Each domain represents a specific "Intent" and is governed by a specific Agent Role.

**Top-Level Space: `Nostra Prime`**

| Sub-Space (Domain) | Intent | BMAD Role | Primary Artifacts |
| :--- | :--- | :--- | :--- |
| **01. Research** | **Discovery** | **Analyst** (Mary) | `ResearchInitiative`, `Brief`, `Sentiment` |
| **02. Planning** | **Definition** | **PM** (John) | `ImplementationPlan`, `Requirement`, `Epics` |
| **03. Engineering** | **Implementation** | **Dev** (Amelia) | `PullRequest`, `Codebase` |
| **04. Library** | **Design** | **Architect** (Winston) | `Schema`, `Component`, `DesignToken` |
| **05. Quality** | **Verification** | **Tester** (Murat) | `TestResult`, `Report`, `Metrics` |
| **06. Operations** | **Process** | **Scrum Master** (Bob) | `Proposal`, `Decision`, `Role` |

### 3.1. Categorization & Labels
To link these domains, we use a shared **Label Taxonomy** (`#development`, `#bug`, `#feature`, `#urgent`).
*   **The #development Stream**: A cross-cutting "Stream" view that sits alongside the Spaces menu. It aggregates all events (commits, research notes, test passes) tagged with `#development` or occurring within `Nostra Prime`.

## 4. UI Design & Components Plan

### 4.1. Dashboard Views (The Stats)
Every Space and Sub-Space must include a **Dashboard Module**:
*   **Metrics**: "Open Issues", "Velocity", "Test Coverage %".
*   **Progress**: Visual bars showing completion of `Epics` or `Initiatives`.
*   **Activity**: Sparklines of recent contributions.

### 4.2. The Testing Module
A dedicated module (likely in the `Quality` domain but accessible globally) to view:
*   **Test Suites**: Browser tests, Unit tests, and E2E flows.
*   **Agent Evaluations**: Logs of Agent performance and success rates.
*   **Reports**: Artifacts generated by the `Tester` agent.

### 4.3. The Bimodal Interface
*   **Nostra View (Territory)**: The heavy-duty editors (Checklists, Code View, Graph View).
*   **Cortex View (HUD)**: The overlay for next actions, agent delegation (`/delegate`), and the `#development` stream.

## 5. Agent Capabilities & Invocation (The Playbook)

We implement the **BMAD "Agent-as-Code"** pattern (`research/017`).

### 5.1. The "Playbook" Mapping
We map user Intents to BMAD Paths:
*   **Intent: "Refine Idea"** -> **Path**: `Analyst/refine-brief` -> **Artifact**: `Brief.md`
*   **Intent: "Plan Feature"** -> **Path**: `PM/create-specs` -> **Artifact**: `Requirements.md`
*   **Intent: "Verify Code"** -> **Path**: `Tester/run-suite` -> **Artifact**: `TestResult.md`

### 5.2. Invocation Strategies
1.  **Passive (The Watchers)**: Agents scan the `Stream` for events matching their Role (e.g., `Tester` sees `PullRequest.Created`).
2.  **Active (The Commander)**: User types `/delegate` in Cortex.
3.  **Workflow (The Engine)**: Moving a card on the Kanban board triggers a defined Workflow (e.g., "Ready for Review" -> auto-assigns `Reviewer`).

## 6. Initial Implementation Steps
1.  **Bootstrap Structure**: Create the `Nostra Prime` space with the 6 Sub-Spaces.
2.  **Define Playbook**: Map the BMAD file structures (e.g., `Start/Step1/Step2`) to `ProcessTemplates` in Nostra.
3.  **Deploy Dashboard**: Create the initial "Stats" component for the `Research` domain.
